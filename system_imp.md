3.3.1 Functional Requirements

User Management:
Customers can register, log in, and log out.
Admins can log in to access a separate dashboard for managing operations.
Order Management:
Customers can place orders for water refills or new bottled water units.
Customers can specify delivery address, quantity, and preferred schedule.
Admin can view all orders, filter by status, and update order progress (e.g., “Pending,” “In Delivery,” “Completed”).
Delivery Management:
Admin can assign delivery staff and monitor order completion.


Customers receive updates once their orders are dispatched or delivered.



Inventory Management:
Admin can monitor available water containers, bottles, and refilling stocks.


Low-stock notifications can be displayed to alert the admin for restocking.


Payment Management:
Customers can select payment methods such as cash on delivery or GCash/Maya.


Admin can verify and record completed payments.


Notification System:
The system sends order confirmation and delivery status notifications to customers.


Admin can send alerts for order delays or availability issues.




Reporting and Analytics:
Admin can view total sales, completed deliveries, and most frequent customers.


The system displays graphical statistics using charts for easy monitoring.


3.3.2 Non-Functional Requirements

Usability: The interface is designed to be clean and simple, suitable for all age groups.
Performance: The system handles multiple concurrent users and order submissions efficiently.
Security: User passwords and payment data are securely encrypted.
Reliability: The system ensures order data and delivery records are consistently saved and retrievable.
Scalability: The system is designed for future enhancements such as adding more delivery areas or multiple admin accounts.
Availability: The web-based system is accessible anytime through a browser and is compatible with mobile devices.
